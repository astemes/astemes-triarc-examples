Customer With Phone.lvclass:Helper Action.vi

The specific action implemented in the helper loop. Multiple different actions may be supported by using the Message and Message Data input to separate them. Any number of helper loops may run in parallell running the same or different actions.

The helper loop rate is controlled by the Loop Perid output of this method.

If this method throws an error, the helper loop will stop. The error will be sent to the error handler configured for the process. To stop it intentionally, the Stop Helper Loop.vi should be used.

One caveat with the helper loop is that it will have a copy of the object attributes as they were when the helper loop was started. If the attributes are updated, the helper loop will need to be restarted to refresh attributes.
