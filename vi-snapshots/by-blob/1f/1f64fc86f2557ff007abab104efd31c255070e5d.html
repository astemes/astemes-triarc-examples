Test.lvlib:Demo Process.lvclass:Handle Messages.vi

This is the core VI handling incomming messages to the process. This VI should be overridden to add functionality to the process or override functionality implemented by a parent. Any message not handled should be sent to the parent method using the default case.

Each state should be atomic and states should in general avoid enqueueing new messages from within this method, as there might be other messages interleaved in the queue from the outside. For cyclic iteration of an action, using a Helper Loop is recommended.
