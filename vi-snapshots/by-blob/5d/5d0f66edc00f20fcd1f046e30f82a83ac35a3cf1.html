Customer.lvclass:Async Action.vi

The specific action implemented in the asynchronous action. Multiple different actions may be supported by using the Message and Message Data input to separate them. Any number of actions may run in parallell running the same or different actions. If this method throws an error, the error will be sent to the error handler configured for the process.

Care needs to be taken so that this action is cancelled if the process is stopped. This may be detected by e.g. observing the Lifecycle: Stopped/Finished broadcast or polling the Get Lifecycle State.vi.
